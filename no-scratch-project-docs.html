🚀 20-Second Elevator Pitch

No Scratch is a free iPhone app for UK National Lottery scratchcard players. It shows every scratchcard currently on sale, with real images, exactly how many top prizes are left, and a visual progress bar so you know at a glance whether a card is still worth buying. Power users pay once to unlock daily email alerts — they get notified when new cards launch or when top prizes are about to run out. Free to download, ad-supported, with a one-time in-app purchase to go ad-free and unlock alerts.

💡 Origin & Problem Statement

The project started as a simple Python command-line script that scraped the National Lottery prize page to look up scratchcard information by game ID. The core insight was:

  • The National Lottery publishes how many top prizes remain for every scratchcard
  • Most players have no idea this information exists
  • Buying a card where the top prize has already been claimed is a waste of money
  • No existing app surfaced this information in a simple, visual way

From that Python script, the project evolved into a full iOS app with cloud backend, analytics, email alerts, and App Store submission — built entirely through an iterative conversation with Claude.

Key insight: The National Lottery is legally required to publish remaining prize data. This is public information that most players simply don't know how to find.

🐍 Flask Backend (app.py)

The entire backend lives in a single app.py file deployed to Render. It handles five responsibilities:

1. Prize Data Scraping

Scrapes national-lottery.com/national-lottery-scratchcard-prizes — a public table of all active scratchcards with remaining top prizes. Uses requests and BeautifulSoup4 with browser-like headers to avoid bot detection.

2. Prize Totals API

Calls the National Lottery's internal CMS API to get allTopPrizes and leftTopPrizes for each card. This data is merged with the scraped prize page data and returned to the iOS app.

GET https://api-dfe.national-lottery.co.uk/cms-proxy/scratchcardList
    ?%40jcr:uuid=1cbcc086-3710-47dd-ba65-7e58365efbfa

3. API Endpoints

EndpointMethodDescription
/api/cardsGETReturns all current scratchcards with prize data
/api/logPOSTLogs a card view event for analytics
/api/subscribePOSTAdds email to subscriber list after purchase
/api/unsubscribeGETRemoves email from subscriber list
/api/analyticsGETPassword-protected analytics dashboard
/api/test-alertGETManually trigger the daily alert check

4. Database (SQLite)

Two tables auto-created on startup:

card_views     -- Analytics: timestamp, card_name, game_no, city, country, device
subscribers    -- Alerts: email, purchase_date, active flag
card_snapshots -- Daily snapshot for change detection

5. Background Threads

Two background threads start on server launch:

  • Prize data prefetch — calls the NL CMS API at startup so first request is fast
  • Alert scheduler — sleeps until 8am UTC, runs daily alert check, sleeps again

Demo Mode

Run with python3 app.py --demo to use hardcoded sample data — useful for testing without hitting the lottery site.

📱 iOS App (Swift)

Card.swift — Data Model

Defines the Card struct conforming to Identifiable and Codable. Custom init(from decoder:) uses decodeIfPresent with fallbacks so missing/null fields never crash the app. Key computed properties:

  • remainingCount — parses remaining string to Int
  • isAlmostGone — true if 3 or fewer prizes left
  • isLow — true if 10 or fewer prizes left
  • remainingPercentage — leftPrizes / totalPrizes for progress bar

NetworkManager.swift — API + Images

The NetworkManager class (ObservableObject) handles all network calls. Key feature: images are hardcoded — a static dictionary maps game numbers to direct CDN URLs extracted from the National Lottery CMS API. This means images load instantly without any server call.

private static let imageMap: [String: String] = {
    let base = "https://assets.national-lottery.co.uk/.imaging/focalpoint/Landscape-3-2/300x"
    return [
        "1495": base + "/dam/jcr:d81a3620.../100tripler.jpg",
        "1479": base + "/dam/jcr:abdcc056.../100kamfay-2025.jpg",
        // ... 40 cards total
    ]
}()

PurchaseManager.swift — StoreKit 2

Handles the one-time in-app purchase using Apple's StoreKit 2 API. Purchase state persisted in UserDefaults so it survives app restarts. Includes restore purchases functionality (required by Apple).

LocationManager.swift — CoreLocation

Requests "when in use" location permission. Reverse geocodes the last known location to city + country level only — never stores precise coordinates. Used solely for analytics.

CardListView.swift — Main Screen

The main grid screen. Uses LazyVGrid with two columns. Includes price filter chips (horizontal ScrollView), search bar, loading state, error state, and banner ad at the bottom for free users. Each card tap logs to analytics via simultaneousGesture.

DisclaimerView.swift — Gambling Disclaimer

Shown once on first launch. Covers: app is information only, 18+ requirement, BeGambleAware link, independence from Allwyn/National Lottery. Acceptance stored in UserDefaults.

Free Tier Limitation

Cold starts: On the free tier, if no requests hit the server for 15 minutes it "spins down". The next request takes ~30 seconds to wake up. The iOS app shows "First load may take up to 30 seconds" to manage user expectation. Upgrade to paid tier (£7/month) to keep it always awake.

📧 Email Alert System

The alert system runs entirely on the Flask backend. No third-party service beyond Resend for email delivery.

How It Works

1

User Purchases Pro

StoreKit purchase completes. UpgradeView shows email entry field.

2

Email Registered

App POSTs email to /api/subscribe. Stored in SQLite. Welcome email sent via Resend.

3

Daily at 8am UTC

Background thread wakes up. Scrapes current card data. Compares with yesterday's snapshot stored in card_snapshots table.

4

Change Detection

New card (game_no not in yesterday's data) → alert. Top prizes ≤ 3 → alert. Less than 20% of prizes remaining → alert.

5

Email Sent

HTML email built with three sections: New Cards, Almost Gone, Running Low. Sent to all active subscribers via Resend API.

Resend

Resend (resend.com) is the email delivery service. Free tier allows 3,000 emails/month. The API is a simple POST request:

POST https://api.resend.com/emails
Authorization: Bearer re_xxxxxxxxxxxx
{
  "from": "alerts@yourdomain.com",
  "to": ["subscriber@email.com"],
  "subject": "🎰 No Scratch — Daily Prize Alert",
  "html": "..."
}

Unsubscribe

Every alert email contains an unsubscribe link: /api/unsubscribe?email=xxx which sets active = 0 in the database. Shows a simple confirmation webpage.

Basic analytics built in-house — no third-party tracking SDK. Data stays on your own server.

What's Tracked

  • Card name — which scratchcard was tapped
  • Game number — the official NL game ID
  • City & Country — approximate location from CoreLocation reverse geocoding
  • Device — iPhone model type
  • Timestamp — UTC datetime

Viewing Analytics

Password-protected JSON endpoint:

https://scratchy-6inm.onrender.com/api/analytics?key=noscrath2024

🗂️ Data Sources

Prize Remaining Data

Scraped daily from the public National Lottery prizes page. This is legally published data that Allwyn is required to maintain.

https://www.national-lottery.com/national-lottery-scratchcard-prizes

Prize Totals + Card Metadata

Fetched from the National Lottery's internal CMS API (discovered via browser DevTools network inspection). Returns allTopPrizes, leftTopPrizes, card names, and image JCR IDs.

https://api-dfe.national-lottery.co.uk/cms-proxy/scratchcardList
    ?%40jcr:uuid=1cbcc086-3710-47dd-ba65-7e58365efbfa

Card Images

Served directly from the National Lottery's Magnolia CMS / Akamai CDN. URLs are hardcoded in NetworkManager.swift using the JCR content IDs extracted from the CMS API. Format:

https://assets.national-lottery.co.uk/.imaging/focalpoint/Landscape-3-2/300x/dam/jcr:{UUID}/{filename}.jpg

Note: The app is not affiliated with or endorsed by Allwyn or The National Lottery. All data is publicly available. The app's disclaimer makes this clear to users.
